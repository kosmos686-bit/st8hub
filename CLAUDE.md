# CLAUDE.md — ST8 AI Hub

## О проекте

**ST8 AI Hub** — это одностраничный CRM и генератор AI-предложений, оформленный как Telegram Mini App. Приложение полностью клиентское: нет бэкенда, нет сборщика, нет пакетного менеджера. Весь код находится в одном файле: `index .html` (в имени файла есть пробел перед расширением — это намеренно и важно сохранить).

**Бизнес-контекст:** Внутренний инструмент продаж компании ST8 AI (Самара). Используется для ведения B2B-воронки в сегментах HoReCa и enterprise, а также для генерации персонализированных коммерческих предложений (КП) и скриптов звонков через API Anthropic Claude.

## Стек технологий

- **HTML5 / CSS3 / Vanilla JavaScript (ES5)** — без транспиляции и фреймворков
- **Telegram Web App SDK** — загружается с `https://telegram.org/js/telegram-web-app.js`
- **Google Fonts** — Montserrat (насыщенность 500–900)
- **Anthropic Claude API** — прямой fetch из браузера на `https://api.anthropic.com/v1/messages`
  - Модель: `claude-sonnet-4-20250514`
  - Максимум токенов: 1500
  - Аутентификация: заголовок `x-api-key` (вводит пользователь), `anthropic-dangerous-direct-browser-access: true`
- **localStorage браузера** — единственный слой хранения данных

## Ключевые соглашения

### Язык
- Весь интерфейс, данные клиентов и комментарии — **на русском**.
- Имена функций и переменных в JavaScript — **на английском**.

### Стиль JavaScript
- **Только ES5** — `var`, без `let`/`const`, без стрелочных функций, без шаблонных строк, без модулей.
- **Глобальное состояние** — весь стейт хранится в переменных верхнего уровня.
- **Императивный DOM** — полный перерендер через `innerHTML` (никакого virtual DOM).
- **Инлайн-обработчики** — `onclick="funcName()"` в HTML-атрибутах — основной паттерн.
- Линтеров и форматеров нет.

### Стиль CSS
- Весь CSS — инлайн в блоке `<style>` в `<head>`.
- Короткие аббревиатурные классы: `.hdr`, `.pg`, `.sc`, `.nb`, `.fb`, `.inp` и т.д.
- Цвета — хардкодные константы дизайн-системы:
  - Золотой акцент: `#D4A017`
  - Тёмный фон: `#0A0F1A`
  - Поверхность карточек: `#111827`
  - Граница: `#1e2d45`
  - Приглушённый текст: `#6B7FA0`
  - Светлый текст: `#E8EDF5`

### Модель данных — объект клиента

```javascript
{
  id:        <число>,    // Уникальный ID, генерируется через nextId()
  name:      <строка>,   // Название компании
  contact:   <строка>,   // Имя контактного лица
  city:      <строка>,   // Город
  type:      <строка>,   // Тип бизнеса (произвольный текст, напр. "HoReCa / Сеть")
  stage:     <строка>,   // Одно из значений STAGES

}
```

## Справочник ключевых функций

### Рендеринг
- `renderStats()` — обновляет KPI-карточки (кол-во клиентов, суммарный потенциал)
- `renderFilters()` — перестраивает кнопки фильтрации
- `renderCards()` — перестраивает список клиентских карточек (с учётом `curFilter`)
- `renderDet(id)` — рендерит панель деталей выбранного клиента
- `renderPipeline()` — рендерит канбан-доску воронки
- `renderKPLib()` — рендерит библиотеку шаблонов КП
- `render()` — вызывает все функции рендеринга (полное обновление)

### CRUD клиентов
- `openAdd()` — открывает модалку в режиме создания
- `openEdit(id)` — открывает модалку, заполненную данными клиента
- `saveClient()` — создаёт или обновляет клиента, сохраняет в localStorage
- `delClient(id)` — удаляет клиента из массива и сохраняет
- `closeModal()` — скрывает модалку

### Навигация
- `switchTab(name)` — активирует нужную вкладку и её кнопку в навигации
- `setFilter(f)` — устанавливает `curFilter` и перерендеривает карточки
- `goKPGen(id)` — переходит на вкладку КП с предзаполненным контекстом клиента
- `goCall(id)` — переходит на вкладку звонка с предзаполненным контекстом клиента

### AI-функции
- `genKP()` — формирует промпт для КП, отправляет в Claude API, выводит результат в `#kp-out`
- `genCall()` — формирует промпт для скрипта звонка, выводит в `#call-out`

### Вспомогательные функции
- `sc(stage)` — возвращает hex-цвет для стадии воронки
- `pi(priority)` — возвращает эмодзи для приоритета (`🔥` / `📌` / `💤`)
- `fmt(n)` — форматирует число как `"250к ₽"` или `"1.0М ₽"`
- `getTotalPot()` — суммирует потенциал всех клиентов
- `nextId()` — возвращает максимальный существующий ID + 1
- `showToast(msg)` — показывает всплывающее уведомление на 2 секунды

### Управление данными
- `exportData()` — копирует JSON массива `CLIENTS` в буфер обмена
- `importData()` — парсит JSON из поля ввода и добавляет клиентов в `CLIENTS`
- `resetData()` — восстанавливает `CLIENTS` из `DEFAULT_CLIENTS` после подтверждения

## Рабочий процесс разработки

### Внесение изменений
Сборщика нет — изменения в `index .html` вступают в силу сразу после перезагрузки страницы.
1. Редактируй `index .html` напрямую.
2. Открой в браузере (или в превью Telegram Mini App).
3. Никаких `npm install`, серверов или компиляции не нужно — подойдёт `file://` или любой статический HTTP-сервер.

### Деплой
Приложение развёртывается через **GitHub Pages** из ветки `main`. Пуш в `main` автоматически инициирует деплой.

### Git-процесс
- Разработка ведётся в ветках вида `claude/...`.
- Слияние в `main` = деплой.

### Добавление нового поля клиента
1. Добавить поле `<input id="f-<имя>">` в HTML модалки.
2. Прочитать значение в `saveClient()`.
3. Отобразить поле в `renderDet()` и/или `renderCards()`.
4. При необходимости обновить записи `DEFAULT_CLIENTS`.

### Добавление новой вкладки
1. Добавить `<button class="nb" data-tab="<имя>">` в `.nav`.
2. Добавить `<div class="tab" id="tab-<имя>">` в `.pg`.
3. Реализовать функцию рендеринга и вызвать её из `render()` или при переключении вкладки.

## Детали интеграции с Claude API

### Паттерн вызова API
```javascript
fetch("https://api.anthropic.com/v1/messages", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "x-api-key": apiKey,
    "anthropic-version": "2023-06-01",
    "anthropic-dangerous-direct-browser-access": "true"
  },
  body: JSON.stringify({
    model: "claude-sonnet-4-20250514",
    max_tokens: 1500,
    messages: [{ role: "user", content: prompt }]
  })
})
```

### Требования к API-ключу
- Должен начинаться с `"sk-"` — проверяется перед вызовом
- Хранится в `localStorage` под ключом `"st8_apikey"`
- Никуда не отправляется, кроме `api.anthropic.com`

## Важные ограничения для AI-ассистентов

1. **Не вводить систему сборки или пакетный менеджер** — архитектура без зависимостей намеренна.
2. **Сохранять ES5-синтаксис** — никаких стрелочных функций, `const`/`let`, шаблонных строк и деструктуризации.
3. **Не разбивать на несколько файлов** — одностраничная архитектура принципиальна.
4. **Сохранять имя файла** `index .html` с пробелом — его изменение сломает GitHub Pages.
5. **Весь интерфейс — только на русском** — это русскоязычный продукт.
6. **Не добавлять бэкенд** — приложение спроектировано как полностью клиентское.
7. **Не шифровать данные в localStorage** — они напрямую используются функциями экспорта/импорта.
8. **Тестировать в мобильном вьюпорте** — приложение оптимизировано для Telegram Mini App (mobile-first).

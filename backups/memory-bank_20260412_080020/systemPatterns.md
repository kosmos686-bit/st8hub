## Архитектурные паттерны

### Интеграции (железно)
- iiko/1С — изолированы, ST8 работает ПОВЕРХ как прозрачный слой
- iiko session token: TTL ~15 мин → обязателен refresh
- MCRM Superkit: нужны api_key + venue_id от клиента
- Каждая внешняя зависимость: retry + timeout + circuit breaker
- Telegram webhook: secret_token обязателен

### Backend паттерны
- ClickHouse — только аналитика
- PostgreSQL — только транзакции (не мешать)
- API версионирование: /api/v1/ с первого эндпоинта
- Внешние вызовы: timeout + retry (tenacity)
- Секреты — только через env, никогда в коде

### Bot паттерны
- Webhook в production (никогда polling)
- Redis для FSM state
- Мессенджер Макс — основной канал (с апреля 2026)
- Telegram — резервный канал
- Триггеры: 24ч/5ч/2ч до визита, ДР в 10:00 МСК (-15%), "давно не были" через 30 дней

### CV паттерны (ОДИСпро)
- Минимум 500 изображений на класс
- Метрики: accuracy ≥ 95%, latency < 500ms, false negative < 2%
- Export: ONNX для edge-инференса
- Права на модели: ST8-AI

### 152-ФЗ
- Персональные данные — только на российских серверах
- Персональные данные — не в логах (только id)
- Телефоны — хешировать в БД

# ST8-AI — Decision Log

## 2026-04 | Memory Bank: единый для ST8-AI

**Контекст:** Рассматривали отдельные Memory Bank под каждый продукт (ST8 Resto, Unikfood, Хокку).
**Решение:** Единый Memory Bank для всей компании ST8-AI.
**Почему:** Агенты работают по всем продуктам сразу. Стандарты, пайплайн, реквизиты — общие. Разделение создаёт дублирование и рассинхронизацию.

## 2026-04 | Мессенджер Макс — основной канал

**Контекст:** Telegram остаётся, но появились ограничения.
**Решение:** Мессенджер Макс — основной канал с апреля 2026. Все новые боты поддерживают оба канала. Telegram — резервный.
**Почему:** Суверенный российский мессенджер, нет рисков блокировки.

## 2026-03 | iiko и 1С — архитектура интеграции

**Контекст:** Клиенты используют iiko и 1С одновременно. Эти системы не интегрируются между собой.
**Решение:** ST8 работает как прозрачный слой ПОВЕРХ обеих систем. Двойной импорт в БД ST8. Никакого обхода POS-системы.
**Почему:** Клиенты не готовы менять POS. ST8 = надстройка, не замена.

## 2026-02 | Оплата 50/30/20

**Контекст:** Нужна единая схема оплаты для всех договоров.
**Решение:** 50% предоплата / 30% после milestone / 20% после приёмки.
**Почему:** Защищает от незакрытых проектов. Финальный платёж = рычаг приёмки.

## 2026-02 | ClickHouse ≠ PostgreSQL

**Контекст:** Первые версии смешивали транзакционную и аналитическую нагрузку.
**Решение:** ClickHouse — только аналитика. PostgreSQL — только транзакции. Не мешать.
**Почему:** Производительность. ClickHouse не подходит для OLTP.
